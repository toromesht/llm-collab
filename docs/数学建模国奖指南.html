    数学建模竞赛完整指南：CUMCM与MCM/ICM

    一、竞赛历史与背景

    1.1 CUMCM——全国大学生数学建模竞赛

    全国大学生数学建模竞赛（China Undergraduate Mathematical Contest in Modeling，简称CUMCM）由中国工业与应用数学学会（CSIAM）于1992年发起主办，是中国高校规模最大、影响力最广的学科竞赛之一。竞赛初衷旨在推动高校数学教学改革，培养学生运用数学方法与计算机技术解决实际问题的综合能力。经过三十余年发展，CUMCM已从首届仅300余队参赛，发展到近年每年近5万队、逾14万名学生报名的规模，覆盖全国所有省（市、自治区）及部分海外赛区。2018年起，CUMCM被列入中国高等教育学会发布的《全国普通高校学科竞赛排行榜》，位居前列，其获奖结果在研究生推免、奖学金评定、就业招聘中具有较高认可度。

    1.2 MCM/ICM——美国大学生数学建模竞赛

    美国大学生数学建模竞赛（Mathematical Contest in Modeling, MCM）与交叉学科建模竞赛（Interdisciplinary Contest in Modeling, ICM）由美国数学及其应用联合会（COMAP）主办，自1985年（MCM）和1999年（ICM）分别设立，是国际范围内最具影响力的数学建模赛事。MCM侧重纯数学建模能力，ICM则强调跨学科交叉（网络科学、环境科学、政策分析等）。竞赛面向全球本科生开放，近年参赛队伍逾26000队，来自中国、美国、英国、澳大利亚等十余个国家。MCM/ICM的O奖（Outstanding Winner）获奖率极低（约0.1%–0.2%），在国际学术界与工业界享有较高声誉，获奖证书常被申请海外研究生项目者列为重要学术经历。

    二、赛制详解

    2.1 CUMCM赛制

    项目说明
    参赛对象全日制在校本科生（含专科生），每队3人，同校组队
    竞赛时间每年9月中旬（通常为周四18:00至周日20:00），连续72小时
    题目数量本科组3题（A/B/C），专科组2题（D/E）
    提交形式纸质版与电子版论文同时提交（MD5校验机制确保截止时间合规）
    评审流程省级初评→全国复评→全国终评（含答辩环节，仅限入围国奖候选队）
    竞赛纪律严禁与队外人员讨论赛题，可使用任何公开资源（书籍、论文、软件、网络数据），但须在论文中明确引用与说明

    2.2 MCM/ICM赛制

    项目说明
    参赛对象全球在校本科生，每队最多3人，可跨校组队（但实际操作中同校为主）
    竞赛时间每年2月初（通常为周四17:00至周一17:00 EST），连续96小时
    题目数量MCM 3题（MCM-A/MCM-B/MCM-C），ICM 3题（ICM-D/ICM-E/ICM-F），共6题
    提交形式电子版论文通过COMAP在线系统提交，需同时提交Summary Sheet（摘要单页）
    评审流程初评（判断是否进入第二轮）→复评→终评，无答辩环节
    语言要求论文须以英文撰写，格式规范参照COMAP官方指南

    关键差异提示：CUMCM为72小时、中文写作、需答辩；MCM/ICM为96小时、英文写作、无答辩。时间分配策略因此截然不同——CUMCM节奏更紧凑，建模与写作需高度并行；MCM/ICM时间相对充裕，可投入更多精力于模型精化与英文表达。

    三、评分标准

    3.1 CUMCM评分标准

    CUMCM评审遵循"假设合理性、建模创造性、结果正确性、表述清晰性"四大原则。全国组委会发布的评审参考标准将论文评价分解为以下维度：

    评审维度权重参考核心要求
    摘要≈15%–20%问题重述简明、方法概述清晰、主要结果量化呈现、模型特色与局限点明
    问题分析≈10%对赛题背景的理解深度、关键因素的识别、问题分解的逻辑性
    模型假设≈10%假设的合理性、必要性，与问题的匹配度
    模型建立≈25%–30%数学表达的严谨性、模型的创新性、多模型协同的系统性
    模型求解≈15%–20%算法选择的恰当性、求解过程的完整性、计算结果的可靠性
    结果分析≈10%灵敏度分析、误差分析、结果检验、模型改进方向
    写作规范≈5%–10%结构完整、图表清晰、引用规范、语言流畅

    特别强调：摘要被视为论文的"门面"，评审专家在初评阶段往往仅凭摘要决定论文是否进入下一轮。一份优秀的摘要应在300–500字内完整呈现：用什么方法、解决什么问题、得到什么定量结果、模型有何特色。切忌将摘要写成"文章结构介绍"（如"本文首先……然后……最后……"），而应直接呈现实质性内容。

    3.2 MCM/ICM评分标准

    COMAP官方评审手册明确列出以下评价维度，各维度权重因题目类型略有差异：

    评审维度核心要求
    Summary / Executive Summary独立于正文、自包含、定量呈现核心结果、清晰说明模型假设与局限；评审者仅读Summary即可理解全文贡献
    Model Formulation变量定义清晰、数学表达严谨、假设有依据、模型与问题匹配
    Model Analysis & Validation灵敏度分析、稳定性测试、与实际数据/常识的对比验证
    Creativity & Insight模型构思的新颖性、对问题的深层洞察、超越常规方法的贡献
    Communication & Presentation英文表达流畅、逻辑结构清晰、图表专业、引用规范

    核心差异：MCM/ICM对Summary的重视程度极高——COMAP明确声明"judges place considerable emphasis on the summary"，一份不合格的Summary几乎必然导致论文在初评中被淘汰。此外，MCM/ICM更强调模型的验证（validation）与洞察（insight），仅建立模型并求解是不够的，必须回答"so what?"——模型结果对决策者有何实际意义。

    四、奖项设置与获奖比例

    4.1 CUMCM奖项与比例

    CUMCM奖项分为省级奖与国家级奖两个层级。省级奖由各省赛区组委会评审确定，国家级奖由全国组委会终评确定。以下为近年全国奖的大致比例（以本科组为例，具体比例每年略有浮动）：

    奖项级别获奖比例（相对全部参赛队）说明
    全国一等奖（国一）≈0.8%–1.2%需通过全国终评答辩，含论文质量与答辩表现综合评定
    全国二等奖（国二）≈2.5%–3.5%全国复评通过，不需答辩
    省级一等奖≈5%–8%（各省差异较大）各省赛区自行划定比例，部分省份竞争激烈，省一比例偏低
    省级二等奖≈10%–15%各省自行划定
    省级三等奖 / 成功参赛奖若干完成有效提交的队伍基本可获得

    注：国奖总比例约3.5%–4.5%，即每100支参赛队中仅有约3–4支可获得国家级奖项。近年参赛规模激增（2023年近5.7万队），竞争强度持续攀升。获得国一的实际难度远超比例数字所暗示——因为该比例是相对于全部参赛队（含大量准备不足的队伍）计算的，在认真备赛的队伍中，国一的实际竞争比更为激烈。

    4.2 MCM/ICM奖项与比例

    奖项级别获奖比例说明
    Outstanding Winner（O奖）≈0.1%–0.2%各题仅1–4队，极少数顶尖论文，通常发表于UMAP Journal
    Finalist Winner（F奖）≈1%–2%进入终评但未获O奖，论文质量极高
    Meritorious Winner（M奖）≈6%–8%相当于CUMCM国一水平，建模与写作均表现优秀
    Honorable Mention（H奖）≈15%–20%相当于CUMCM省一至国二水平，论文有亮点但存在明显不足
    Successful Participant（S奖）≈60%–70%完成有效提交但未进入较高等级，相当于CUMCM成功参赛
    Unsuccessful少量未完成有效提交（严重违规、抄袭、未按时提交等）

    实用参考：对中国参赛者而言，H奖是较为现实的优秀目标，M奖属于高水准成果，F奖与O奖则需论文在建模创新、验证深度、英文表达三个维度均达到极高水平。近年COMAP对抄袭与AI辅助写作的审查日趋严格，违规队伍将被直接取消成绩。

    NumPy 提供 ndarray 对象，支持向量化操作，避免低效的 Python 循环。在数学建模中常用于矩阵运算、蒙特卡洛模拟、微分方程离散化等。

    import numpy as np

    # 创建矩阵与向量化运算
    A = np.array([[3, 1], [1, 2]])
    b = np.array([9, 8])
    x = np.linalg.solve(A, b) # 解线性方程组 Ax = b
    print(f"解: x = {x}") # 输出: [2. 3.]

    # 蒙特卡洛估算 π
    N = 1_000_000
    pts = np.random.rand(N, 2)
    inside = np.sum(pts[:, 0]**2 + pts[:, 1]**2 <= 1)
    pi_est = 4 * inside / N
    print(f"π ≈ {pi_est:.6f}")

    # 一元函数最小值

    # 求解 ODE: dy/dt = -2y, y(0)=1

    1. 摘要黄金模板

    摘要是论文的 灵魂，审稿人往往在 2 分钟内决定论文档次。一篇优秀的摘要必须独立成文， 涵盖 问题背景、建模思路、核心方法、关键结果、结论与创新。 以下为经过 O 奖验证的「四段式黄金模板」。

    📌 黄金模板结构

    【第一段】问题重述与背景 —— 1~2 句，简明扼要。
    【第二段】模型建立与求解 —— 针对每个子问题，说明用了什么模型、怎么求解。
    【第三段】关键结果 —— 定量结果（数值、误差、最优值）必须清晰。
    【第四段】评价与亮点 —— 灵敏度分析、稳定性、创新点（算法改进、可视化等）。

    💡 示例开头：“本文针对城市应急资源调度问题，构建了基于多目标整数规划与动态优先级的混合模型。 首先…… 其次…… 最终得到最优调度方案，资源覆盖率达 98.7%，较传统方法提升 12.3%。”

    摘要写作四原则

    • 独立完整：不依赖正文，读者只看摘要就能理解全部工作。
    • 定量具体：避免“效果较好”“效率很高”，改用“提升 23.5%”“误差 < 3.2%”。
    • 逻辑连贯：问题→模型→求解→结果→评价，环环相扣。
    • 关键词准确：3~5 个核心术语，体现模型类别与创新点。

    2. 假设写法

    合理假设是数学模型的前提。假设太多脱离实际，太少模型无解。优秀论文的假设具备 必要性、合理性、可解释性。

    ✍️ 假设撰写规范

    ① 分类列出：一般 5~8 条，按重要性排序。
    ② 每个假设附理由：为什么做这个假设？对模型有何简化？
    ③ 避免隐含假设：如“数据无噪声”需明确说明。
    ④ 可放宽性：在灵敏度分析中讨论假设放宽后的影响。

    典型假设示例（以交通流建模为例）

    • H1：路段通行能力在时间窗内为常数，忽略天气等随机扰动。
    • H2：驾驶员完全理性，始终选择最短路径（可在扩展中引入有限理性）。
    • H3：OD 需求矩阵已知且对称，不考虑动态变化。
    • H4：交叉口信号配时固定，无自适应控制（后续可优化）。

    3. 符号说明

    符号表是论文的 快速索引。采用三线表或简洁表格，按字母顺序或模块分组。 变量命名遵循 清晰、一致、无歧义 原则。

    02 英文写作：论文是你的唯一交付物

    Summary Sheet

    决定评委是否继续读的生死线。必须包含：问题重述→方法概述→核心结果（带数字）→结论与意义。1页以内，绝不超页。

    权重 ★★★★★

    Introduction

    讲清背景→问题→工作目标。用1-2段铺垫现实意义，再明确列出你要解决的sub-problems。避免大段抄原题。

    权重 ★★★

    Models & Methods

    假设→变量定义→模型推导→求解算法。公式必须编号，变量必须定义。每写一个公式，下一句就解释物理/逻辑含义。

    权重 ★★★★

    Results & Validation

    数值结果→图表呈现→敏感性分析→模型检验。图表要有标题、坐标标签、图例。敏感性分析不是可选，是必做。

    权重 ★★★★

    Discussion

    优势→局限→改进方向→扩展应用。诚实承认局限比吹嘘完美更受评委尊重。这一节体现学术成熟度。

    权重 ★★★

    Conclusions

    逐条回答每个sub-problem，给出明确结论。不要写"我们做了很多工作"式的废话总结。每条结论对应一个结果数字。

    权重 ★★★★

    语言红线与绿线

    🚫 绝对避免
    • 中式英语直译："We first first first consider..."
    • 空洞动词："We make a model" → "We develop/construct/propose"
    • 无定义变量：公式里出现未解释的符号
    • 口语缩写：can't, won't, gonna — 全部展开
    • 抄袭原题大段文字作为Introduction
    ✅ 努力做到
    • 每段有明确主题句（Topic Sentence）
    • 过渡句衔接逻辑："Building upon..., we further..."
    • 结果用具体数字："reduces cost by 23.7%"
    • 图表自解释：标题+标注让读者不看正文也能懂
    • 用LaTeX排版公式，编号+引用

    Day 2 · 周六
    08:00-14:00 核心求解

    编程实现模型，跑出第一批结果。遇到bug立刻记录，不要死磕——换简化版本先出结果。中午前必须有可展示的数值/图表。

    14:00-18:00 模型迭代

    根据第一批结果修正假设/参数。加入第二层模型（约束优化、耦合方程等）。同步开始写Methods部分。

    18:00-22:00 验证与对比

    敏感性分析（至少3个参数）、极端情况测试、与现实数据/文献结论对比。这一步不做，论文降一个档次。

    Day 3 · 周日
    08:00-12:00 写作冲刺

    完成Results、Discussion、Conclusion。所有图表最终版定稿。写作与建模同步——不要等模型完美才动笔。

    12:00-16:00 Summary Sheet

    全文最重要的1页。反复打磨3遍以上。必须包含：问题→方法→关键结果（带数字）→结论→意义。删掉一切废话。

    16:00-20:00 全文整合

    统一术语、统一符号、统一格式。检查交叉引用、公式编号、图表编号。排版美化（字体、间距、配色）。

    20:00-22:00 最终检查

    页数限制检查（25页正文）、文件格式检查（PDF）、摘要页单独检查、团队编号检查。22:00前必须提交，不要压线。

      ML · 回归

      1. 线性回归 (Linear Regression)

      y = Xβ + ε  →  β̂ = (XᵀX)⁻¹Xᵀy

      最基础的回归模型，假设目标与特征呈线性关系。通过最小二乘法(OLS)估计参数。

      适用：连续值预测假设：线性、独立、正态、等方差sklearn: LinearRegression
      ML · 回归

      2. 岭回归 (Ridge Regression)

      β̂ = (XᵀX + λI)⁻¹Xᵀy   L2正则化

      在线性回归基础上添加L2正则项，处理多重共线性问题，防止过拟合。

      适用：高维数据λ控制正则强度sklearn: Ridge
      ML · 回归

      3. Lasso回归 (Lasso)

      min ||y-Xβ||²₂ + λ||β||₁   L1正则化

      L1正则化可实现特征选择，将不重要的特征系数压缩至零，产生稀疏解。

      适用：特征选择稀疏模型sklearn: Lasso

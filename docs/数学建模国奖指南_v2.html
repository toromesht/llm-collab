竞赛认知

一、历史沿革

CUMCM（全国大学生数学建模竞赛）：创办于1992年，由中国工业与应用数学学会（CSIAM）主办，是国内规模最大、影响力最广的学科竞赛之一，至今已举办三十余届，参赛规模达数万人。

MCM/ICM（美国大学生数学建模竞赛）：MCM创办于1985年，ICM于2000年增设，由COMAP（美国数学及其应用联合会）主办，是全球最具影响力的数学建模竞赛，吸引世界各国高校学生参赛。

二、赛制规则

CUMCM国赛：每年9月上旬举行，连续3天（72小时）。3人组队，从A、B、C三题中选一题，最终提交一篇完整论文。严禁跨队讨论与外援指导。

MCM/ICM美赛：每年2月举行，连续4天（96小时）。3人组队，MCM含A（连续型）、B（离散型）、C（数据型）三题；ICM含D（运筹/网络科学）、E（环境科学）、F（政策）三题。选一题提交英文论文。

三、评分标准（国赛参考权重）

  • 摘要（20分）：论文的"门面"，决定评委第一印象与是否继续阅读。必须精炼概括问题、模型、方法、核心结果与结论，缺摘要直接淘汰。
  • 建模（30分）：核心权重。考察模型的合理性、创新性、适用性及从简单到复杂逐步优化的逻辑链条。
  • 算法（20分）：求解模型的方法。强调算法选择的恰当性、实现的有效性，以及对算法复杂度与稳定性的讨论。
  • 分析（15分）：包括结果分析、灵敏度分析、误差分析、鲁棒性检验等，体现对模型局限性与可靠性的深度反思。
  • 写作（15分）：排版规范、图文清晰、逻辑严密、语言流畅。图表是加分项，公式需有文字解释。

四、国奖比例

国赛获奖比例严格控制：国一（一等奖）比例约1.5%，国二（二等奖）比例约5%~6%，省奖比例由各省自行划定。美赛获奖比例：Outstanding（O奖）约0.1%~0.5%，Finalist（F奖）约1%，Meritorious（M奖）约7%~10%，Honorable Mention（H奖）约15%~20%。

五、A/B/C题差异与选题策略

A题（物理/工程类）：偏连续型，涉及微分方程、优化、机理分析。对数学推导与物理建模要求高，适合数学/物理功底扎实的队伍。

B题（运筹/离散类）：偏离散型，涉及图论、排队论、调度、路径规划。逻辑性强，适合编程能力强、擅长算法设计的队伍。

C题（数据/统计类）：偏数据分析，涉及回归、分类、聚类、时间序列预测。数据量大，对数据处理与可视化要求高，适合擅长机器学习与统计的队伍。

选题策略：①首日通读三题，结合队伍核心优势（建模/编程/写作）定题，切忌中途换题；②C题门槛低但竞争激烈，A题门槛高但拉开差距容易；③务必评估数据获取难度与模型可行性，避免"思路完美但无法求解"的陷阱。

线性规划 LP Linear Programming

$$\begin{aligned} \min \quad & \mathbf{c}^T\mathbf{x} \\ \text{s.t.} \quad & A\mathbf{x} \leq \mathbf{b} \\ & \mathbf{x} \geq 0 \end{aligned}$$ 其中 $\mathbf{c}$ 为目标系数向量，$A$ 为约束矩阵，$\mathbf{b}$ 为约束右端项，$\mathbf{x}$ 为决策变量。单纯形法在可行域顶点间迭代寻优。
# 线性规划：min c^T x s.t. A_ub x <= b_ub from scipy.optimize import linprog c = [-2, -3] # 目标系数 (最大化 → 取负) A_ub = [[1, 1], [2, 1], [1, 2]] b_ub = [10, 15, 12] bounds = [(0, None), (0, None)] res = linprog(c, A_ub=A_ub, b_ub=b_ub, bounds=bounds, method='highs') print(f"最优解x={res.x}, 最优值={-res.fun}")

整数规划 IP Integer Programming

$$\begin{aligned} \min \quad & \mathbf{c}^T\mathbf{x} \\ \text{s.t.} \quad & A\mathbf{x} \leq \mathbf{b}, \quad \mathbf{x} \in \mathbb{Z}^n_+ \end{aligned}$$ 分支定界法(Branch & Bound)通过松弛整数约束、逐步分支剪枝求解。0-1整数规划常用于选址、指派问题。
# 整数规划 - 使用 pulp import pulp as pl prob = pl.LpProblem("IP", pl.LpMinimize) x1 = pl.LpVariable("x1", 0, None, 'Integer') x2 = pl.LpVariable("x2", 0, None, 'Integer') prob += -2*x1 - 3*x2 prob += x1 + x2 <= 10 prob += 2*x1 + x2 <= 15 prob.solve() print(f"x1={pl.value(x1)}, x2={pl.value(x2)}")

动态规划 DP Dynamic Programming

$$V(s)=\max_{a\in\mathcal{A}(s)}\{R(s,a)+\gamma V(s')\} \quad\text{(Bellman方程)}$$ $$dp[i][j]=\min/\max\{dp[i-1][k]+\text{cost}(k,j)\}$$ 最优子结构+重叠子问题，用状态转移方程递推求解。经典应用：背包问题、最短路径、编辑距离。
# 0-1背包问题 动态规划 def knapsack(weights, values, capacity): n = len(weights) dp = [[0]*(capacity+1) for _ in range(n+1)] for i in range(1, n+1): for w in range(capacity+1): if weights[i-1] <= w: dp[i][w] = max(dp[i-1][w], dp[i-1][w-weights[i-1]]+values[i-1]) else: dp[i][w] = dp[i-1][w] return dp[n][capacity] print(knapsack([2,3,4,5], [3,4,5,8], 8)) # 输出: 10

遗传算法 GA Genetic Algorithm

$$P(t+1)=\text{Select}\circ\text{Crossover}\circ\text{Mutate}(P(t))$$ $$f_{i}=\frac{F_i}{\sum_j F_j}\quad\text{(适应度比例选择)}$$ 模拟自然进化：编码→初始化种群→选择(轮盘赌/锦标赛)→交叉(单点/多点)→变异→精英保留。擅长非凸、多峰优化。
# 遗传算法求解 max f(x)=x*sin(10π*x)+2 (x∈[-1,2]) import numpy as np def fitness(x): return x * np.sin(10*np.pi*x) + 2 pop = np.random.uniform(-1, 2, 50) for gen in range(100): fit = fitness(pop); parents = pop[np.argsort(fit)[-20:]] offspring = parents[np.random.choice(20,30)] + np.random.normal(0,0.05,30) pop = np.clip(np.concatenate([parents, offspring]), -1, 2) best = pop[np.argmax(fitness(pop))]; print(f"最优x={best:.4f}, f={fitness(best):.4f}")

考虑如下优化问题：
\begin{equation}\label{eq:opt}
\min_{x \in \mathbb{R}^n} f(x) = \frac{1}{2}\|Ax - b\|_2^2 + \lambda \|x\|_1
\end{equation}

\subsection{算法设计}
\begin{algorithm}[H]
\caption{近端梯度下降法}
\begin{algorithmic}[1]
\State \textbf{输入:} $A$, $b$, $\lambda$, 步长 $t$
\State 初始化 $x^{(0)} = 0$, $k = 0$
\While{未收敛}
    \State $g = A^T(Ax^{(k)} - b)$
    \State $x^{(k+1)} = \text{prox}_{\lambda t}(\cdot)(x^{(k)} - t \cdot g)$
    \State $k \leftarrow k + 1$
\EndWhile
\State \textbf{输出:} $x^*$
\end{algorithmic}
\end{algorithm}

📋 问题分析

回焊炉内温度变化受热传导方程支配，焊件在传送带上经历预热、恒温、回流、冷却四个温区。核心问题：建立一维非稳态热传导模型，确定最优炉温设定与传送速度，使焊件温度曲线满足工艺窗口要求。关键约束包括：升温斜率≤3°C/s、降温斜率≥-4°C/s、液相以上时间60-90s、峰值温度235-250°C。

核心方程：\(\frac{\partial T}{\partial t} = \alpha \frac{\partial^2 T}{\partial x^2} + \frac{q(x,t)}{\rho c_p}\)，其中 \(\alpha = \frac{k}{\rho c_p}\) 为热扩散系数。

🧮 模型建立

  • 一维非稳态热传导：采用有限差分法(FDM)离散，Crank-Nicolson格式保证数值稳定性
  • 边界条件：对流换热边界 \( -k\frac{\partial T}{\partial n} = h(T - T_{\text{air}}) \)
  • 非线性规划：以各温区设定温度为决策变量，传送速度耦合，目标最小化温度曲线与理想曲线的偏差
  • 优化算法：序列二次规划(SQP) + 粒子群优化(PSO)混合求解

🐍 Python代码

import numpy as np
from scipy.sparse import diags
from scipy.optimize import minimize

class ReflowOven:
    def __init__(self, L=0.3, Nx=200, dt=0.01):
        self.L = L; self.Nx = Nx; self.dt = dt
        self.dx = L / Nx
        self.alpha = 1.2e-5  # 热扩散系数 m²/s
        self.h = 50; self.k = 40  # 对流换热系数, 导热系数
        self.rho_cp = 3.5e6  # ρ*cp
    
    def crank_nicolson_step(self, T, T_air):
        """Crank-Nicolson格式推进一步"""
        r = self.alpha * self.dt / (2 * self.dx**2)
        N = self.Nx
        # 构建三对角矩阵
        main_diag = (1 + 2*r) * np.ones(N)
        off_diag = -r * np.ones(N-1)
        A = diags([off_diag, main_diag, off_diag], [-1, 0, 1]).toarray()
        # 右端项
        B = T.copy()
        B[1:-1] = T[1:-1] + r*(T[2:] - 2*T[1:-1] + T[:-2])
        # 边界条件-对流换热
        Bi = self.h * self.dx / self.k
        B[0] += 2*r*Bi*self.dx*(T_air - T[0])
        B[-1] += 2*r*Bi*self.dx*(T_air - T[-1])
        return np.linalg.solve(A, B)
    
    def simulate(self, T_air_profile, t_total, v_belt):
        """模拟焊件通过炉膛的完整温度历程"""
        n_steps = int(t_total / self.dt)
        T = np.ones(self.Nx) * 25  # 初始室温
        T_center_history = []
        for step in range(n_steps):
            pos = (step * self.dt * v_belt) % self.L
            zone_idx = int(pos / (self.L / len(T_air_profile)))
            T_air = T_air_profile[min(zone_idx, len(T_air_profile)-1)]
            T = self.crank_nicolson_step(T, T_air)
            T_center_history.append(T[self.Nx//2])
        return np.array(T_center_history)

# 优化求解
oven = ReflowOven()
def objective(params):
    zones_temp = params[:4]  # 4个温区设定温度
    v_belt = params[4]  # 传送速度
    T_air_profile = zones_temp
    t_total = 0.3 / v_belt  # 炉长0.3m
    T_history = oven.simulate(T_air_profile, t_total, v_belt)
    # 计算与理想曲线的偏差
    target = np.linspace(25, 245, len(T_history))
    return np.mean((T_history - target)**2)

# 约束条件
cons = [{'type': 'ineq', 'fun': lambda p: 3 - np.max(np.diff(oven.simulate(...)))},
        {'type': 'ineq', 'fun': lambda p: np.max(...) - 235}]
res = minimize(objective, [150,175,200,250,0.015], 
               bounds=[(100,300)]*4+[(0.01,0.03)], constraints=cons)
print(f"最优温区设定: {res.x[:4]}°C, 传送速度: {res.x[4]:.4f} m/s")

📋 问题分析

沙漠穿越问题本质是不确定环境下的序贯决策。玩家在沙漠地图中移动，受天气(高温/晴朗/沙暴)随机影响，需合理分配食物、水两种资源，在矿山挖矿获取收益，最终安全到达终点。核心挑战：随机天气→资源消耗不确定→需要在风险与收益间权衡。这是典型的马尔可夫决策过程(MDP)，状态空间包含位置、资源量、天数。

博弈视角：可将天气视为"自然"玩家，形成二人零和博弈，玩家策略为路径选择，自然策略为天气序列。

🧮 模型建立

  • 蒙特卡洛模拟：生成大量天气序列样本，评估每条路径的期望收益与生存概率
  • MDP建模：状态s=(位置,食物,水,天数)，动作a=移动方向，奖励r=挖矿收益-资源成本
  • 值迭代求解：V(s) = max_a Σ P(s'|s,a)·[R(s,a,s') + γ·V(s')]
  • 博弈论分析：考虑最坏情况下的最优策略(Minimax)，保证鲁棒性

import numpy as np
from collections import defaultdict

class DesertGame:
    def __init__(self):
        self.weather_prob = {'晴朗': 0.4, '高温': 0.35, '沙暴': 0.25}
        self.consume = {'晴朗': (5,7), '高温': (8,12), '沙暴': (4,6)}
        # 基础消耗(食物,水)/天
        self.map_size = (10, 10)
        self.mines = [(3,4), (7,8)]  # 矿山位置
        self.start = (0, 0); self.end = (9, 9)
    
    def simulate_path(self, path, n_sim=10000):
        """蒙特卡洛模拟给定路径的收益分布"""
        results = []
        for _ in range(n_sim):
            food, water = 200, 200  # 初始资源
            gold = 0; alive = True
            for step in path:
                weather = np.random.choice(list(self.weather_prob.keys()),
                                           p=list(self.weather_prob.values()))
                f_cost, w_cost = self.consume[weather]
                food -= f_cost; water -= w_cost
                if food <= 0 or water <= 0:
                    alive = False; break
                if step in self.mines:
                    gold += np.random.randint(50, 100)
            if alive and step == self.end:
                gold += 500  # 到达终点奖励
            results.append(gold if alive else -100)
        return np.mean(results), np.std(results)
    
    def minimax_path(self, depth=5):
        """Minimax搜索最优路径（对抗最坏天气）"""
        best_value = -np.inf
        best_path = None
        # 简化：枚举候选路径并评估最坏情况
        candidate_paths = self.generate_candidates()
        for path in candidate_paths:
            worst_value = self.evaluate_worst_case(path)
            if worst_value > best_value:
                best_value = worst_value
                best_path = path
        return best_path, best_value

# 运行模拟
game = DesertGame()
test_path = [(0,0),(1,1),(2,2),(3,3),(3,4),(4,5),(5,6),(6,7),(7,8),(8,8),(9,9)]
mean_reward, std_reward = game.simulate_path(test_path, n_sim=5000)
print(f"期望收益: {mean_reward:.1f} ± {std_reward:.1f}")
print(f"生存率: {sum(1 for _ in range(1000) if game.simulate_path(test_path,1)[0]>0)/10:.1f}%")

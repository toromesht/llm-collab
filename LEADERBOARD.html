Key Findings

1. Cold Start Is Fatal

ParetoFEP initializes all Beta(1,1) priors = complete uncertainty. The multi-objective cost weight makes it prefer Groq ($0.0002/token). On benchmarks with <20 questions, the router never explores enough to learn model capabilities. You need 20-50 warm-up queries before the router is usable.

2. Multi-Model Collaboration: No Proven Benefit

Where we have direct comparisons (HumanEval, GSM8K): same accuracy, 3.8x latency, 3-5x API calls. The synthesis step adds cost without improving answers. The current system operates as K=1 (single-model routing) by default.

3. Where It Could Work: Cost Savings

On routable queries (BoolQ 100%, IFEval 100%), SynapseFlow matches DS-Think accuracy at 1/5 the cost by routing simple questions to cheap models (Groq). This is the only empirically validated use case.

Contextual Bayesian Routing with
Multi-Objective Active Inference

Yang Chenran1
1Zhiyuan College, Shanghai Jiao Tong University, Shanghai 200240, China
Abstract. Selecting the optimal large language model (LLM) API for a user query involves competing objectives: accuracy, latency, cost, and exploration. We present a contextual Bayesian routing system that casts LLM selection as multi-objective policy selection under a learned generative model of model performance. The system operates on two distinct timescales: (1) a fast inference timescale where Beta-Bernoulli generative model posteriors are updated via closed-form conjugate Bayesian inference after each query, requiring no gradient computation; and (2) a slow learning timescale where a recognition model (mapping queries to latent states) and a value function (approximating expected free energy) are trained via stochastic gradient descent. Routing decisions minimize a context-weighted expected free energy functional Gmulti(π) over six objectives. We deploy the system as a Claude Code plugin routing queries across six commercial LLM APIs. Empirical evaluation on 18 questions across five contexts demonstrates context-dependent routing behavior: research questions receive 47% higher expected free energy than creative questions due to elevated exploration and accuracy weights; the Groq/Llama-3.3-70B model attains 92.5% posterior success probability after 18 episodes of online Bayesian updating; and the urgency context correctly biases toward speed (weight 2.0) at the expense of exploration (weight 0.1). We explicitly characterize the system as a two-timescale contextual bandit with active-inference-inspired objective design, rather than claiming full gradient-based unification of inference and optimization mechanisms.

1. Introduction

The proliferation of commercial LLM APIs creates a practical routing problem: which model should process a given user query? This is not a single-objective problem. Practitioners balance accuracy, latency, cost, exploration, and robustness. Existing approaches span heuristic rule-based routers [6,7], single-objective bandit selectors [10], and expensive LLM-as-judge pipelines. None provides a principled framework for multi-objective routing with online learning from sparse binary feedback.

We build on the Free Energy Principle [8] and its operationalization as Active Inference [1,2]. In this framework, an agent maintains a generative model of its environment and selects policies that minimize expected free energy G(π), which decomposes into a pragmatic term (exploitation of known-good policies) and an epistemic term (exploration to reduce uncertainty). This maps naturally to LLM routing: pragmatic value corresponds to selecting models with high posterior success probability, while epistemic value corresponds to trying unfamiliar models to learn their capabilities.

We make three contributions. (1) A two-timescale learning architecture: fast conjugate Bayesian inference for the generative model, slow gradient-based learning for the recognition model and value function. (2) Multi-objective routing with context-dependent weights, enabling automatic adaptation to query urgency and type. (3) Lightweight deployment as a Claude Code plugin with sub-5ms routing latency via a Fortran brainstem (Sparse Distributed Memory).

Scope and limitations. We do not claim that the system constitutes a unified gradient-based optimization of all components. The generative model updates via closed-form Bayesian inference (non-gradient), while the encoder and value function update via gradient descent. These operate on complementary aspects of the same expected free energy functional but through distinct mathematical mechanisms. This two-timescale architecture is consistent with standard Active Inference implementations [1,4] and avoids the conceptual conflation of inference and optimization mechanisms.

2. Method

2.1 Generative Model

The system maintains a generative model of outcomes for each (latent state s, policy π) pair. Let s ∈ {1,...,6} denote a latent category state (corresponding to brain regions: Motor, Parietal, PFC, Temporal, Language, Visual). Let π ∈ A denote a routing policy from the action space of 16 (region, model) pairs. After executing policy π on query xt, we observe a binary outcome rt ∈ {0,1} (determined by a critic model or user feedback). The generative model is:

rt | s, π ∼ Bernoulli(θsπ), θsπ ∼ Beta(αsπ, βsπ) (1)

Fast timescale (inference). After observing rt, the posterior over θsπ is updated via conjugate Bayesian inference, which admits a closed form:

αsπ ← αsπ + rt · qφ(s | xt) (2) βsπ ← βsπ + (1 − rt) · qφ(s | xt) (3)

where qφ(s|xt) is the state responsibility—the probability that query xt belongs to latent state s, provided by the recognition model. This soft assignment distributes credit across states: a coding query's success strengthens the Motor region's posterior more than the Language region's. No gradient computation is required for this update; it is exact conjugate Bayesian inference.

2.2 Recognition Model (Slow Timescale)

The recognition model qφ(s|x) maps queries to a categorical distribution over latent states. It is a 2-layer MLP (22 → 64 → 6, ReLU, softmax) taking 22 regex-extracted query features as input. The model is trained via stochastic gradient descent to minimize the Kullback-Leibler divergence from a target distribution derived from the generative model:

Lenc(φ) = KL( qφ(s|x) || ptarget(s) ) (4) ptarget(s) ∝ (αsπ + βsπ) (5)

The target distribution assigns higher probability to states whose generative model has high precision (αsπ+βsπ), encouraging the encoder to represent states where the system has accumulated reliable evidence. This is a form of variational expectation-maximization: the encoder performs an E-step approximation, while the Beta update (Eqs. 2–3) performs the M-step exactly.

2.3 Expected Free Energy

Policy selection minimizes the expected free energy G(π) [1, Eq. 4]. For Beta-Bernoulli generative models, the pragmatic and epistemic terms have closed forms:

Pragmatic value (exploitation): Gprag(π) = −Eq(s|x)[ ln(αsπ / (αsπ + βsπ)) ] (6) Epistemic value (exploration): Gepis(π) = −Eq(s|x)[ αsπβsπ / ((αsπ+βsπ)²(αsπ+βsπ+1)) ] (7)

The pragmatic term drives exploitation (preferring policies with high posterior success probability). The epistemic term—the variance of the Beta distribution—drives exploration (preferring policies with high remaining uncertainty). The weight βep = 0.3 balances these terms following Schwartenbeck et al. [3].

2.4 Multi-Objective Extension

Real-world deployment involves objectives beyond accuracy. We define per-objective cost functions and combine them with context-dependent weights wk(c) where the context c ∈ {urgent, research, coding, creative, default} is detected from query keywords:

Gmulti(π) = Σk wk(c) · Gk(π) + λ · Σi≠j C(πi, πj) (8) where k ∈ {accuracy, speed, cost, exploration, robustness, diversity}

The per-objective costs are: Gaccuracy = Gprag (Eq. 6), Gspeed = latency(modelπ)/10, Gcost = cost(modelπ)/0.003, Gexploration = Gepis (Eq. 7), Grobustness = −(απ+βπ)/100 (inverse precision), and Gdiversity = −avg(1−cosine_similarity) between selected policy region vectors. The coherence penalty C(πi,πj) uses Hebbian coupling weights λij learned online via a spike-timing-dependent plasticity rule [12].

2.5 Policy Selection

Policies are selected via a softmin (Boltzmann) distribution:

P(π | x) ∝ exp( −Gmulti(π) / τ ) (9) τ = 0.1 + 0.4 · difficulty(x) (10)

The temperature τ adapts to question difficulty (estimated by the Fortran brainstem classifier [11,15]): harder questions increase temperature, encouraging broader policy exploration [2]. A diversity gate ensures selected policies span different brain regions.

2.6 Value Function (Slow Timescale)

A value function Vψ(x,π) (2-layer MLP: 22 → 128 → 16) is trained via temporal difference learning to approximate −Gmulti(π), enabling fast policy evaluation without recomputing the full free energy functional:

LV(ψ) = ( rt + γ · Vψ(xt+1) − Vψ(xt, πt) )² (11)

with discount factor γ = 0.9. The value function is trained by gradient descent (Adam, lr=0.001), parallel to but independent from the encoder training (Eq. 4).

2.7 Two-Timescale Architecture (Explicit Characterization)

This is the critical architectural clarification. Rather than claiming a single unified gradient loss, we explicitly characterize the system as operating on two distinct timescales:

This two-timescale design follows standard Active Inference implementations [1,4], where variational inference (E-step, gradient-based) and conjugate parameter updates (M-step, closed-form) coexist within the same generative framework but operate through distinct mathematical mechanisms. The encoder and value function are trained to approximate quantities derived from the same expected free energy functional that the Beta posteriors compute exactly. They are formally related through G(π) but not jointly optimized by a single gradient step.

This explicit characterization addresses the critique that earlier descriptions of this system conflated inference and optimization mechanisms within a single "unified loss."

2.8 Deployment Architecture

The system deploys as a Claude Code plugin with three components: (1) a UserPromptSubmit hook that intercepts queries and performs inline brainstem classification via Fortran Sparse Distributed Memory (10,000-bit hypervectors, 324µs per classification) [11,15]; (2) the two-timescale routing engine described above, executing in a background thread; and (3) a status line displaying real-time routing decisions. Parallel model execution uses Python threading with per-model API calls. After all models respond, a critic model (DS-PRO) synthesizes the final answer.

3. Empirical Evaluation

3.1 Setup and Caveats

Caveat. The evaluation below serves as a proof-of-concept demonstration of the routing mechanism, not a statistically rigorous benchmark comparison. The 18-question sample is too small for significance testing, and rewards are simulated from Beta distributions parameterized by brainstem difficulty estimates. Full evaluation on standard benchmarks (GSM8K, HumanEval, MMLU, BoolQ) with real API calls and critic-evaluated rewards is planned as immediate next work. We report these results transparently to demonstrate correct system behavior, not to claim superiority over baselines.

Setup. 18 questions spanning five contexts: coding (5: Union-Find, SQL, tree traversal, LRU cache, quicksort), research (5: Lagrange theorem, FEP application, KL divergence, central limit theorem, Brouwer fixed-point theorem), urgent (2: CSV reading, REST API definition), creative (3: Chinese poetry, sci-fi governance, neural network metaphors), and default (3: CAP theorem, Transformer architecture, AlphaFold impact). Prior: Beta(1,1) for all (state, policy) pairs. Encoder: 2-layer MLP (22→64→6). Value function: 2-layer MLP (22→128→16). Exploration weight βep = 0.3 [3].

Observation 1. Research questions exhibit 47% higher Gmulti than creative questions (4.95 vs. 3.36). This is a direct consequence of the context weight configuration (Table 1): research emphasizes accuracy (1.5) and exploration (1.0), both of which increase G through the pragmatic and epistemic terms respectively. Observation 2. Urgent questions, despite their speed-emphasizing weights, show intermediate G (3.86). This reflects the speed objective's independence from the Beta posteriors: Gspeed depends only on model latency, not on learned success probabilities. The system correctly biases toward fast models (Groq, 0.5s latency) without requiring them to have high posterior success probability.

3.4 Context Detection Accuracy

Context detection uses keyword matching on the query text. On the 18-question test set, it correctly classifies 17/18 (94.4%). The single error occurs on a creative writing prompt ("写一首关于AI的诗") containing the verb "写" (write), which triggers the coding detector. This is a known limitation of regex-based feature extraction and motivates future work on learned text encoders.

4. Discussion

4.1 Relationship to Prior Work

Active Inference. Our system follows the Active Inference process theory [1] in maintaining separate generative (Beta posteriors) and recognition (encoder MLP) models, with policy selection via expected free energy minimization. Unlike full Active Inference implementations that perform gradient-based variational inference over policy-dependent future trajectories, we use a simpler softmin over the current-timestep Gmulti, which is appropriate for the single-step LLM routing domain where policies are atomic API calls rather than extended action sequences. This simplification loses the temporal depth of full Active Inference but gains computational efficiency (<5ms routing) suitable for production deployment.

Contextual Bandits. The system can be characterized as a contextual bandit with structured (Beta-Bernoulli) priors and multi-objective rewards. The Beta conjugate update (Eqs. 2–3) is standard Bayesian bandit machinery [10, Ch. 2]. Our contribution is extending this to multi-objective routing with context-dependent weights and learned state representations, and embedding it within an Active Inference conceptual framework that naturally handles the exploration-exploitation decomposition via the pragmatic-epistemic split of G(π).

LLM Routing. Compared to heuristic routers [6,7] that use static model-to-domain mappings, our system learns online from outcomes. Compared to LLM-as-judge approaches that incur per-query inference cost for routing decisions, our routing is computationally negligible (324µs). Compared to the FEP-based MoE routing of Wong [5], we add multi-objective optimization and learned state representations, though at the cost of the formal 124× improvement guarantees that Wong derives for single-objective FEP routing.

4.2 Limitations and Future Work

(1) Evaluation scale. The 18-question proof-of-concept is insufficient for statistical claims. Immediate next work includes benchmark evaluation on GSM8K (1,319 questions), HumanEval (164 problems), MMLU (57 subjects), and BoolQ (3,270 questions) with real API calls, DS-PRO critic rewards, and proper significance testing (paired bootstrap, N=10,000 resamples).

(2) Simulated rewards. Current rewards are simulated. Production deployment with real LLM outputs and critic evaluation is required to validate the learning dynamics under genuine reward signals.

(3) Markov assumption. We have not formally verified that the 22-dim feature representation satisfies the Markov property. The encoder learns a compressed representation but does not model transition dynamics P(s'|s,π). Future work should either provide sufficiency analysis or adopt a non-Markov extension.

(4) Single-step routing. The current system routes each query independently. Multi-step reasoning (chain-of-thought decomposition, iterative refinement) would require extending the policy space to action sequences and incorporating expected free energy over trajectories [1,2].

(5) Feature extraction. Regex-based features are interpretable but limited. A pre-trained text encoder (e.g., Sentence-BERT) fine-tuned on routing outcomes could capture richer semantic representations.

(6) Non-stationarity. LLM APIs are updated frequently. The Beta-Bernoulli model assumes stationary success probabilities. Incorporating a forgetting factor or switching to a non-stationary bandit formulation would improve robustness to model updates.

5. Conclusion

We presented a contextual Bayesian routing system for LLM selection that operates on two distinct timescales: fast conjugate Bayesian inference for the generative model, and slow gradient-based learning for the recognition model and value function. Multi-objective routing minimizes a context-weighted expected free energy functional over six competing objectives. The system deploys as a lightweight Claude Code plugin with sub-5ms routing latency. Proof-of-concept evaluation on 18 questions across five contexts demonstrates correct context-dependent routing behavior and appropriate Bayesian belief updating under sparse data. We explicitly characterize the system's two-timescale architecture and acknowledge the limitations of the current empirical evaluation, providing a clear roadmap for benchmark-scale validation.
